### Задание 1. Исследование "Гидра зависимостей"

1. Создайте проект через Spring Initializr, выбрав только один starter: `Spring Web`.
2. Откройте файл `pom.xml` (Maven) или `build.gradle` (Gradle) и посмотрите список зависимостей. Обратите внимание, что вы добавили только одну строку кода.
3. Разверните дерево зависимостей в вашей IDE (в IntelliJ IDEA это вкладка Maven -> Dependencies). Посчитайте, сколько реальных библиотек подтянул этот один starter. Напишите краткий вывод, почему это удобнее ручного управления.

### Задание 2. Анализ автоконфигурации

1. Добавьте в ваш `application.properties` строку `debug=true`.
2. Запустите приложение и скопируйте из консоли раздел `Positive matches`.
3. Найдите в этом списке `DispatcherServletAutoConfiguration`. Объясните (опираясь на отчет), при каком условии Spring Boot решил создать этот компонент.
4. **Критерий выполнения:** вы должны четко указать строку из отчета, содержащую `@ConditionalOnClass` или аналогичное условие.
