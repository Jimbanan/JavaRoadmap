### Задание 1: «Система управления задачами (Task Tracker)»

Создайте REST-контроллер для управления списком задач.

1. Разработайте класс предметной области`Task`(поля:`id`(String),`title`(String),`completed`(boolean)).
2. Создайте`TaskController`и настройте базовый путь`/api/tasks`на уровне класса.
3. Реализуйте метод`GET`для получения всех задач.
4. Реализуйте метод`POST`для добавления новой задачи.
5. Реализуйте метод`DELETE`для удаления задачи по`id`.
6. **Критерий выполнения:**Приложение должно запускаться, а эндпоинты должны быть доступны через инструменты тестирования API (например, HTTPie, cURL или Postman) и корректно работать с памятью (ArrayList).

### Задание 2: «Инвентаризация склада с контролем статусов»

Добавьте продвинутую логику обновления данных.

1. Реализуйте метод`PUT`для обновления задачи.
2. **Логика:**если задача с таким`id`существует, обновите её и верните статус`200 OK`. Если задачи нет, создайте новую и верните статус`201 CREATED`.
3. Используйте`ResponseEntity`для формирования ответа.
4. **Критерий выполнения:**При успешном обновлении существующего объекта в заголовках ответа должен приходить статус`200`, а при создании нового —`201`. Это должно быть подтверждено тестом через HTTP-клиент.
