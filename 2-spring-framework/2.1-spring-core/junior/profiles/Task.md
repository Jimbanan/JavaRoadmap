### Задание 1: "Гибкий логгер"

**Контекст:** Ты пишешь систему обработки заказов.

1. Создай интерфейс `OrderLogger` с методом `logOrder(String orderId)`.
2. Создай реализацию `ConsoleOrderLogger` (просто `System.out`), которая работает только в профиле `dev`.
3. Создай реализацию `FileOrderLogger` (имитируй запись в файл), которая работает только в профиле `prod`.
4. Создай реализацию `MockOrderLogger` (пустой метод), которая работает в профиле `default`.
5. Попробуй запустить приложение с разными ключами `-Dspring.profiles.active` и убедись, что внедряется нужный бин.

### Задание 2: "Тестирование интеграции"

**Контекст:** Нужно проверить, что в тестовом окружении используются правильные настройки.

1. Создай файл `application-test.properties` и пропиши в нем свойство `app.upload.path=/tmp/test-uploads`.
2. Создай сервис, который читает это свойство через `@Value`.
3. Напиши JUnit-тест с аннотацией `@ActiveProfiles("test")`, который проверяет (через `assertEquals`), что сервис действительно получил путь для тестов, а не значение по умолчанию.
