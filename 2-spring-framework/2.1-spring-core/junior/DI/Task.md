### Задание 1: Система уведомлений (Логика DI)

**Контекст:** Вы разрабатываете систему обработки заказов в интернет-магазине. **Задача:**

1. Создайте интерфейс `NotificationService`.
2. Реализуйте две версии: `EmailNotificationService` (с `@Primary`) и `SmsNotificationService`.
3. Создайте класс `OrderProcessor`, который принимает `NotificationService` через конструктор.
4. В методе `processOrder()` вызывайте уведомление.
5. Проверьте, что внедряется именно Email-версия. Затем, используя `@Qualifier`, переключитесь на SMS-версию без изменения логики `OrderProcessor`.

### Задание 2: Жизненный цикл и Scope

**Контекст:** Вашему приложению нужно логировать время каждой сессии пользователя, но сам логгер должен быть уникальным для каждой операции. **Задача:**

1. Создайте класс `CustomLogger`. Сделайте его `@Scope("prototype")`.
2. Добавьте метод с `@PostConstruct`, который выводит в консоль время создания бина.
3. Внедрите этот логгер в синглтон-сервис `AdminService`.
4. Вызовите метод `AdminService` несколько раз. Посмотрите на логи. **Вопрос на раздумье:** Почему время в логгере не меняется при каждом вызове метода синглтона, хотя логгер — Prototype? (Это поможет вам закрепить понимание того, как Spring внедряет зависимости один раз при создании синглтона).
